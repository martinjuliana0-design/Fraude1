# Fraude1
---
## Descripción del proyecto
---
Este proyecto tiene como objetivo analizar un conjunto de transacciones financieras con el fin de comprender el comportamiento del fraude desde una perspectiva descriptiva. A través del uso de SQL Server Management Studio para la exploración de datos y Power BI para la visualización, se identifican patrones temporales y económicos asociados a las transacciones fraudulentas.
El análisis está orientado a un enfoque analítico y exploratorio (EDA), sin aplicar modelos predictivos, simulando el trabajo de un Data Analyst junior en un contexto real de negocio.
---
## Objetivo del proyecto 
---
Analizar el comportamiento de las transacciones fraudulentas para identificar patrones temporales y diferencias en los montos respecto a las transacciones no fraudulentas.
---
## Hipotesis iniciales 
---
1.	El fraude representa un porcentaje bajo del total de transacciones.
2.	El fraude no ocurre de manera uniforme en el tiempo.
3.	Existen bloques temporales con mayor riesgo relativo de fraude.
4.	Las transacciones fraudulentas presentan un comportamiento económico distinto al de las transacciones normales.
---
## Analisis exploratorio de datos 
  # Volumen y proporcion:
  
*El fraude representa una proporción muy pequeña del total de transacciones.
*A pesar de su baja frecuencia, tiene impacto económico.
 
  # Análisis temporal :
  
*El fraude no se distribuye de forma homogénea en el tiempo.
*Se identificaron bloques horarios con mayor concentración de fraudes.
*El análisis de la tasa de fraude (%) permitió detectar periodos de mayor riesgo relativo.
